How it works

Zero config. Three steps.

Step 01

Run one command

npx obolus scan

No install, no API key, no configuration file. Just run it in any terminal. Supports a --since flag (7d, 30d, …) to scope the time window.

Step 02

Reads local history — metadata only

Obolus reads the Claude Code session history already on your disk. It extracts token counts, model, repo, branch, and timestamps only. Your code and prompts never leave your machine.

Step 03

See attributed spend

Get a breakdown by repo, model, branch, and session, with cross-run history and a top-sessions ranking that the native /usage command cannot give you.

Why Obolus

What makes it different

🔒

Privacy-first, offline

Metadata only — never your code or prompts. Runs fully offline against local files. Nothing is sent to any server. Open source so you can verify it yourself.

📊

Beats native /usage

Claude Code's built-in /usage only covers your own machine, with no per-repo breakdown. Obolus adds cross-repo, per-branch and per-session attribution with persistent history — the full picture.

🌐

Cross-vendor by design

Built around a vendor-neutral schema from day one. Claude Code is supported now; Cursor and Codex importers are on the roadmap — so your spend history survives every tool switch.

⚡

Zero friction

No account, no daemon, no Claude API key. One command and you see results. Free and MIT-licensed — use it however you want.

An obol was the small coin the ancient Greeks placed under the tongue to pay Charon, the ferryman across the Styx. Obolus watches the small coins your agents spend before they add up to a fare you never meant to pay.
